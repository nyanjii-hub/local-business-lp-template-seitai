# Chiropractic LP

地域密着型の整体院を想定した、ポートフォリオ用の自主制作ランディングページです。

## Overview

このLPは、整体院を初めて知ったユーザーが「どんな悩みに対応しているのか」「どのように予約できるのか」「安心して相談できそうか」を短時間で判断できるように設計した1ページ構成のWebサイトです。

肩こり・腰痛・姿勢改善などの悩みを訴求し、LINE予約・電話予約・Instagram・店舗情報までを1ページで案内できるテンプレートとして制作しています。

## Project Goal

地域の整体院が「予約につながるホームページ」を持つことを目的とした自主制作LPです。

見た目を整えるだけではなく、初めて訪れるユーザーが不安を減らしながら、サービス内容の理解から予約まで自然に進める導線設計を意識しました。

特に、スマートフォンから閲覧するユーザーが多い店舗サイトを想定し、ファーストビューでの訴求、予約CTA、施術内容、患者様の声、アクセス情報を1ページ内に整理しています。

## Design Thinking

### 想定クライアント

- 地域密着型の整体院
- 個人から小規模事業者
- スマートフォンからのアクセスが多い店舗

### 想定課題

- 初めて訪れたユーザーが「どんな整体院なのか」を短時間で理解できない
- 予約方法が分かりづらく、予約ページへ進まず離脱してしまう可能性がある
- 施術内容や対象となる症状が整理されておらず、自分に合うか判断しづらい

### 改善施策

1. ファーストビューで価値を明確化
   - 肩こり・腰痛・姿勢の悩みなど、どんな人の課題を解決できる整体院なのかを最初に伝え、続きを読む理由を作る

2. 予約導線を分かりやすく配置
   - ファーストビュー、予約セクション、画面下部の固定CTAに予約ボタンを配置し、迷わずLINE予約・電話予約へ進める導線を設計する

3. 情報を整理して安心感を向上
   - コンセプト、施術メニュー、患者様の声、施術風景、店舗情報を順番に配置し、初めて利用する人でもサービス内容を理解しやすい構成にする

### Expected Outcome

本作品は自主制作のため、実際の運用データや改善実績はありません。

一方で、一般的なWebサイト改善の考え方にもとづき、以下のような成果を狙った設計としています。

- ファーストビューでの離脱率低減
- 予約ページへの遷移率向上
- 初回利用者の安心感向上
- サービス内容の理解促進

## Local Development

このリポジトリは静的HTMLで構成されています。

### Browser Preview

`index.html` をブラウザで開くと確認できます。

### Local Server

必要に応じて、任意のローカルサーバーで確認できます。

```bash
python -m http.server 3000
```

ブラウザで `http://localhost:3000` を開いてください。

## Customization

以下の項目を実際の整体院情報に差し替えることで、店舗向けLPとして利用できます。

| 項目 | 検索する文字 | 変更例 |
|---|---|---|
| 院名 | `Your Seitai` / `Your Shop` | `〇〇整体院` |
| 電話番号 | `000-0000-0000` / `0000000000` | `03-1234-5678` / `0312345678` |
| LINE | `@your_line_id` | `@sample_seitai` |
| Instagram | `your_account` | `sample_seitai` |
| 住所 | `東京都中央区銀座1-2-3` | 実際の住所 |
| 施術メニュー | `肩こり改善整体` など | 自院の施術メニュー |
| 料金 | `¥4,800` など | 自院の料金 |
| 営業時間 | `平日 10:00〜19:00` など | 自院の営業時間 |
| 患者様の声 | サンプル文 | 実際の声や掲載許可済みの内容 |

## Google Map

このテンプレートではAPIキー不要の埋め込みURLを使っています。

```html
https://www.google.com/maps?q=東京都中央区銀座1-2-3&output=embed
```

住所部分を整体院の住所に変更してください。道案内ボタン側のURLも同じ住所へ差し替えると自然です。

## Note

本作品はポートフォリオ用の自主制作であり、実際の店舗データや運用実績にもとづくものではありません。

課題設定、改善施策、成果仮説は、一般的なWebサイト改善・LP改善の考え方をもとに設計しています。実際の効果を保証するものではなく、「売上増加」や「予約数増加」などの成果を断定するものではありません。

また、画像はUnsplashの外部画像を使用しています。商用利用時は必要に応じて自院の写真に差し替えてください。患者様の声や評価数はサンプルです。実際に掲載する場合は、掲載許可や引用ルールを確認してください。

医療的な効果を保証する表現は避け、必要に応じて専門家や広告ガイドラインを確認してください。
